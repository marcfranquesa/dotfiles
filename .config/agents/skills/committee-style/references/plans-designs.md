# Plan and Design Reviews

Use this reference for plans, technical designs, roadmaps, product proposals, process changes, project strategies, and execution sequencing.

## Reviewer Roles

- feasibility reviewer;
- simplicity/scope reviewer;
- risk reviewer;
- user-outcome reviewer;
- integration/sequencing reviewer;
- domain reviewer.

## Coverage

Assign reviewers to assumptions, dependencies, milestones, risks, interfaces, users, and rollout or verification paths when those surfaces are distinct.

Tell reviewers to test whether the plan is executable with the current constraints, whether it can be simplified, and which assumptions would change the decision if false.

## Synthesis

Lead with blockers, brittle assumptions, sequencing problems, missing owners or verification, and scope that should be cut or deferred.

Separate disagreements about taste from disagreements that materially change the recommended path.
